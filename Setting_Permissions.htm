Setting Permissions

Set permissions for each role

  1. On the left-hand side menu, point to User Management, and then click Permissions.

  2. Two sections - Roles and Menu Permissions appear.

  3. In the Roles section, under Name, select the required role (highlighted as “1”).

  4. Under Menu Permission, the permissions for the selected role (highlighted as “2”) appear.

  5. To select permissions for a specific role, select the check box before the required permissions, and then click Save.

  6. To deselect permissions for a specific role, clear the check box before the required permission, and then click Save.

    Note:
    Each Role can have limitations on the Menu Permissions.
    For example, an ORBIT ADMIN ROLE can have all the menu permissions, however, the menu permissions for a ORBIT USER ROLE can be limited.
    Admin Role has the highest priority followed by Super user, and the last priority comes for end user roles.

  • Default permissions for Roles can be restricted by selecting or clearing the check boxes next to the options.

Reports

Enabling Reports section, allow users to access reports module.

Reports - Creating Data Report

Enabling Creating Data Report option, allow users to create report with business objects and datasets.

Reports - Creating Analysis

Enabling Creating Analysis option, allow users to create report with business objects and datasets.

Reports - Creating Dashboard

Enabling Creating Dashboard option, allow users to create report with business objects and datasets.

Report - Share

Enabling Share option, allow users to share the report to a user or a role. Share permission helps user to view the data of the report.

Report - Export

Enabling Export option, allow users to export the report data to Excel and other formats with the help of templates.

Report - Generate Public Report Link

Enabling Generate link option, allows user to generate public and private links. With the help of Generate link, an Orbit user can share their reports to the people who do not have access to the Orbit application

Report - Query settings

Enabling Query Setting option, allow users to modify the default query settings of a data report.

Reports - Creating Publisher Report

Enabling Creating Publisher Report option, allow users to create report with business objects and datasets.

Reports - Import Oracle Report(s)

Enabling Import Oracle Report(s) option allow users to import Oracle concurrent programs as Orbit Custom reports.

Reports - Creating Script Report

Enabling Creating Script Report option, allow users to create report with business objects and datasets.

Pre - Execute Block

Enabling Pre-Execute Block option, allows user to access the executive block.

A Pre - execution can be performed at logical model level and report level.

Reports - Creating Custom Report

Enabling Creating Custom Report option, allow users to create report with business objects and datasets.

Reports - Creating Finance Report

Enabling Creating Finance Report option, allow users to create report with GLSense templates.

Allow database functions in expression builder

Enabling Allow database functions in expression builder option, allow users to access evaluate function.

Reports - Show SQL

Enabling Show SQL option, allow users to view the SQL script of the executed report.

Report - Delivery Plans

Enabling Delivery Plans option, allow users to deliver a report with applied conditions through email.

Report - More Widgets

Enable More Widgets option, allow users to create a new custom widget for Analysis Report.

Report - Edit Custom Widgets

Enabling Edit Custom Widgets options, allow users to edit the custom widgets in Analysis Reports.

Report - Delete Custom

Enabling Delete Custom Widgets options, allow users to delete the existing created widgets in Analysis Reports.

Report - Download Custom

Enabling Download Custom Widgets options, allow users to download the created custom widgets from Analysis Reports.

Report - Email Delivery

Enabling Email Delivery option, allow users to deliver the report data through Emails

Report - Allow Insights

Enabling Allow Insights option, allow users to access insight in data report.

An insight report is a child report that originates from a data report. Insight report can have fewer columns compared to its parent report.

Domains

Enabling Domains option, allow users to access domains page.

Domains - Create

Enabling Create option, allow users to create domain from different datasources. 

Domain - Import

Enabling Domain Import option, allow users to import objects from datasource to the physical model.

Domain - Show TestQuery Result

Enabling Show TestQuery Results option, allow users to build an SQL query from the business object and view the data/result of the generated query.

Datasets

Enabling Datasets option, allow users to access datasets page.

DataSets - Create from Query

Enabling Create from Query option, allow users to create a datasets from SQL Query with the respective datasource.

Datasource 

Enabling Datasource option, allow users to create a connection setup to a database from a server. It is used when creating a domain and a query to the database.

Security

Enabling Security option, allow users to provide security to the Business Object.  

Metaobjects

Enabling Meta-Objects option, allow users to apply conditional formats to the columns in the reports.

Finance Cube

Enabling Finance Cube option, allow users to access finance cube page and can also perform to create, edit, delete, share the finance cube, and create cube batch.

User Management - Users

Enabling Users option, allow users to create/registering a user in application

User Management - Groups

Enabling Groups option, allow users to assign different users and different roles to the group.

User Management - Roles

Enabling Roles option, allow users to assign different users and different groups to the roles. 

User Management - Authorities

Enabling Authorities option, allows users to import users, roles, and groups from different datasources.

User Management - Permission

Enabling Permission option, allow users to accesses and restrict different operations in the application.

Monitoring

Enabling Monitoring option, allow users to track the scheduled reports or jobs and the completed processes of reports and Jobs.

Enabling Processes option, allow users to track the completed or ongoing processes list.

Enabling Report Statistics option, allow users to track the report stats.
