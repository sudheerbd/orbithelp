Setting Global Security

To set the Global Security

 

1. In the required logical model, click Data Security.

2. Data Security-<Logical Name>dialog appears.

3. To set the global security, select the Global radio button (highlighted as “1”).

4. The formula box (highlighted as “2”) appears as shown below:

5. In the formula box, click (highlighted as “3”) to create a formula.  

6. Expression Builder (Data Filters) dialog box appears.

5. To create an expression in the expression field, perform the following:

a. Under the Category section, select the required category from the following:

  • Available Objects

  • Functions

 Required category: Available Objects

  • Sub-Category changes based on the selected Category

b. Under the Sub-Category section, select the required sub-category:

 Required Sub-category: AP_INVOICES_ALL

c. When a sub-category is selected, its associated columns appear under “Double click to insert” section.

Required Column: ORG_ID

d. Create an expression in the expression field (highlighted as “4”) as shown below:

e. To save the changes, click Ok.

6. The formula appears in the formula box (highlighted as “5”).

7. To save the created formula, click Ok.                                                                            

8. Create a business object and drag and drop the required columns under New Category.

9. To save the changes to business object, click Save.

10. Assign the required security to the business object, and then click Save.

 

Viewing the applied Global Security in Report

To view the applied global security in a report                 

1. Create a new data report, and then select the business object which has global security applied on it.

2. Drag and drop the required columns from Business Object section to Report Configurator section, and then click Apply.
