Creating an LDAP Authority

To Create a New LDAP Authority

 

  1. On the left-hand side menu, point to  User Management, and then click  Authorities.

  2. Authorities page appears with a list of authorities.

  1. To create a new authority, click New (highlighted as "1") on the top right-hand side of the Authorities page.

  1. New Authority dialog box appears.

  1. In the New Authority dialog, enter the following to create a new authority

    • In the Name box, type the name of the authority.

Example: Name: OPEN-LDAP

    • In the Authority Key box, type the authority key.

Example: Authority Key: LDAP_AUTH_KEY

    • To sync the users/settings on login, select Sync on Login checkbox.

    • In the Description box, type the description for the authority.

  1. Under the Authentication section, in the Authority Type box, type/select the required authority type from the following:

    • Oracle ERP Security

    • LDAP

    • Active Directory

    • PeopleSoft

Example: Required Authority Type: LDAP

Based on the selected authority types, the fields in the Authentication section change.

    • In the Server URL box, type the URL of the Server.

Example: Server URL: ldap://192.168.2.124:389

Note: Here, ldap is the protocol, "192.168.2.124" is the ip, and 389 is the default port number.

    • In the User DN box, type the Username.

Example: User DN: ORBIT8\Administrator

    • In the Password box, type the password.

Example: Password: Welcome@123

    • In the Base DN box, type the Base DN.  

            Base DN: This is a point from where the server starts its search.

  Syntax: dc=orbit, dc=com
