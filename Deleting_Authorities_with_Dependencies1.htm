Deleting Authorities with Dependencies

When deletion is attempted on an authority having a dependency, the authority does not get deleted straightaway. Its associated users and roles must be deleted first to delete an authority.

Scenario

In the below screen, a new authority (highlighted) is created and users are imported from that authority.

When deletion is attempted, it displays the message" This authority cannot be deleted. Some of the users are already imported. Please remove reference objects(users,roles)".

 

This implies that the user imported must be deleted prior to the deletion of  an authority.
