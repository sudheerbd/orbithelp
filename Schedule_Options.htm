Schedule Options

Working with schedule options

 

  1. Under the Reports area, the required folder appears with its associated reports.

  2. Right-click the required report, and then click Schedule.

  3. Schedule - Report dialog box appears.

  4. Schedule tab is in disabled mode, by default.

  5. To enable the Schedule tab, clear Run Immediately check box.

 

  1. To view schedule options, click (highlighted as "1").

 

  1. A list of options appears under options section.

    • Priority

      1. To run the schedules in priority order, select the required value.

Priority options helps when multiple threads are running at same time.

Priority does not define order of execution, it applies when there are more number of jobs than the available number of schedule - work threads. By default, the thread count is 15 and can be controlled from <application.properties> file.

    • Missfire Policy

      1. IGNORE: Never Runs missed schedules

      2. RUNALL: Runs all missed schedules on scheduler startup

      3. RUNONCE: Runs once for all missed schedules (Default)

A misfire policy occurs, when a scheduled job skipped the firing time, because of instance/scheduler being shutdown or in a stand by state.

    • Calendar

      1. list of calendars appears under Calendar

Calendar will determines whether the firing time is included by the calendar or not. If the calendar ends up excluding the trigger then the job will not fire.

When a job is set to scheduled on everyday, can be prevented from firing on holiday by associating the trigger with a calendar.

Calendar dropdown list will be available, when user have calendar under maintenance section. To create a calendar refer Scheduler Calendar.

    • Timezone Offset

      1. Select the required timezone from the dropdown list.

Timezone offset helps the user to specify, on which time zone a job needs to trigger.

Example 2

Here we are standby the scheduler.

  1. To standby the scheduler, navigate to Maintenance window.

  2. On the left-hand side menu, point to   System, and then click Maintenance.

  3. Scheduler page appears with Operations tab.

  4. To standby the scheduler, click Stand By.

  5. A alert message "Are you sure you want to StandBy Scheduler?" appears.

 

  1. To standby the scheduler, click Yes.

  2. To view the schedules status which are already in running, navigate to schedules window.

  3. On the left-hand side menu, point to   Monitoring, and then click Processes.

  4. To view schedules status, click Schedules tab.

  5. The executing schedule status changes to waiting status.
