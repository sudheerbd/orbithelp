Discoverer Migration into ORBIT Analytics

Prior to migration, two data sources need to be configured. One is the connection used for querying discoverer reports, and the other is data source for EUL.

 

The following are the two data sources that are configured here:

·         discoverer

·         discoverer_eul5

 

     

 

 See Create New Data Sources to configure the data sources for discoverer migration into ORBIT Analytics

 

 

Once the data sources are configured, start the discoverer migration process.

 

The user can perform either of the following processes for Discoverer Migration. 

 

a. The user can either export the .eex file from the Desktop administrator and migrate the worksheets. See Migrating Worksheets into ORBIT Analytics section for more information.

 

(OR)

 

b. The user can migrate the business areas and then upload the schedule xml into ORBIT Analytics. Once the domain is created the user can build and view the reports.

See Migrating Business Areas section for more information.

ü Migrating Business Areas

ü Migrating Worksheets
