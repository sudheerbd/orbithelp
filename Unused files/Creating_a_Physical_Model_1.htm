Creating a Physical Model

To create a new Physical Model

 

  1. On the newly created domain page, Domain Hierarchy appears.
  2. Under Domain Hierarchy area, Physical Models and Logical Models (highlighted as “1”) appear.             

   3.    To create a new physical model, right-click Physical Models, and then click New Physical Model

   4.      New Physical Model dialog box appears.

 

                                       

 

 

5.      In the New Physical Model dialog box, perform the following:

·         In the Name box, type the name of the new physical model.

 

      ?     Example: Name: Invoices Physical Model

 

·         In the Description box, type the description for the new physical model.

 

?     Example: Description: Invoices Physical Model

 

·         In the Data Source box, select or type the required existing data source.

 

?     Example: Data Source: Test Data Source

 

(OR)

 

·         Create a new data source.

·         To create a new data source, click Create new datasource .  See Create new datasource.  

·         To save the changes, click Save.

·         To close the dialog box, click Cancel.                                       

 

6.      The newly created physical model appears under Physical Models.

                                        

 

7.      To pull the tables into the newly created physical model, right-click the physical model, and then click Import Objects.

 

8.      Import Object(s) - <Physical Model Name> dialog box appears.

 

9.      In the Select Schema list, click the required option from the list.

 

?     Example: Select Schema Option: APPS

 

10.  In the Name box, type the table/synonym/view name or the search term, and then click Search.

 

?     Example: Search Term: invoices

 

11.  A list of available objects (highlighted as “3”) appears under Available Objects area.

 

    12.  Select the required object, and then click.

 

?     Example: Required Objects: AP_INVOICES

                                                           AP_INVOICES_ALL

 

 

13.  The selected objects appear under Included Objects area.

14.  To import the objects, click Import.

15.  The selected objects appear under the Physical Model as shown below:

                                                            

¥  The above procedure is explained using “Import Objects” option. See other options to import objects below.

 

¥    When the tables are imported from diverse sources, ensure that the columns in the tables have valid data types. If the imported columns do not have valid data types then set them appropriately in the physical model.

¥    If the datatypes for the imported columns are not set appropriately, then the columns would not be available to set as prompts.
