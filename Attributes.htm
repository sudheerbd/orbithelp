Attributes

All the LDAP objects have attributes that take unique or multiple values, these values describe the object characteristics. For example, a user object in LDAP will have attributes such as his first name, last name, email etc.

The name of an attribute is similar to the name of a field in a database. The actual value assigned to the attribute is stored in LDAP.


Orbit supports the attributes with the following values:

  • userNameAttributeValue = "uid"

  • groupNameAttributeValue = "cn"

  • groupRoleAttributeValue = "member"

  • userObjectClassValue = "person"

  • groupObjectClass=”group”

  • groupRoleAttribute=”member”

However, the available attributes can be overwritten using the required attribute value. 

To Change the Attributes

  1. In the Edit Authority dialog, on the Authentication section, click  Attributes.

  2. The Attributes dialog appears.

  1. Using the Attributes section, add the new attribute or delete the existing attribute.

  2. To add a new attribute, click  Add Attribute.

 

  • Initially the “groupObjectClass” had a value “group”. This is now overwritten with the value “groupOfNames”.

  1. To save the attribute, click Ok.
