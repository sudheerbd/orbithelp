Pre Executive Block

Pre-execute block is triggered before the report is actually executed. Pre-execution can be performed at the following two levels:

  • Report Level

  • Logical Model Level

 

Pre-requisites

 

In the Logical Model or Physical Model, the query executor can be either of the following:

  • Oracle ERP

  • Orbit Content

  • Oracle (with prompt init)

Here, the query executor has been taken as Oracle (with prompt init)

 

Exercise 1

 

Setting Pre-execute Block at Report Level

 

To set the pre-execute block at Report Level

1. Open the required report.

2. On the right-hand side, click  Options, and then click Pre Execute Block.

3. Pre Execute Block box appears.

4. In the Pre Execute Block box, type the required information (select/create statements) as shown below:

 

  • Create table AP_INVOICES_ALL1 as select * from AP_INVOICES_ALL

 

 

5. To continue, click Ok.

6. To execute the condition in Pre Execute Block, click Run.

7. The report appears as follows:

8. A table with “AP_INVOICES_ALL1” is created in the database with all the records of “AP_INVOICES_ALL” table.

The above procedure is same for Pre Execute block in Analysis.

Prerequisite: For Orbit_content, the user should have apps intialization.

 

1. Open the required report.

2. On the right-hand side, click  Options, and then click Pre Execute Block.

3. Pre Execute Block box appears.

4. In the Pre Execute Block box, type the required statements as shown below:

 

begin

insert into period_name(period_name)

select period_name from xxfo.RXAP_INVOICE_REGISTER;

commit;

end;

 

   

 

5. To continue, click Ok.

6. To execute the condition in Pre Execute Block, click Run.

7. The report appears as follows:
