Deleting Authorities with Dependencies

When deletion is attempted on an authority having a dependency, the authority does not get deleted straightaway. Its associated users and roles must be deleted first to delete an authority.

Scenario

In the below screen, a new authority "TestAUTH" is created and users are imported from that authority.

 

When deletion is sttempted on TestAUTH, it displays the message" This authority cannot be deleted. Some of the users are already imported. Please remove reference objects(users,roles)".

This implies that the user imported using TestAUTH must be deleted prior to the deletion of  TestAUTH authority.
