Joins

Logical Joins express the relationship between the logical tables. A JOIN is used to combine rows from two or more tables, based on a related column between them.

Joins are primarily classified into the following

 

  • Inner Join
  • Left Outer Join
  • Right Outer Join
  • Full Outer Join

 

 

Inner Join

 

The INNER JOIN selects records that have matching values in both tables.

 

For example, you are running a store and would like to record information about the customers and their orders. Consider two tables –Customers (table 1) and Orders (table 2) which have been joined using an inner join.

Left Outer Join  

The Left Outer Join or Left Join returns all records from the left table, and the matching values from the right table. If there is no match in the right table, the result is NULL.

 

Considering the example of Customers(table1) and Orders(table2) table, the left join returns all the data from the Customers table and returns only the matching records from Orders table. In case, the records of Customers table does not have any matching order details,the left join returns a NULL in its place.

Right Outer Join

 

The RIGHT OUTER JOIN or simply Right Join returns all records from the right table and the matched records from the left table. When there is no match in the left table,the result is NULL.

 

Considering the examples of Orders table (table 1) and Employees table (table2), the right join returns all the data from the Employees table and returns only the matching records from the Orders table. In case the records of Employee table does not have any matching order details, the right join returns a NULL in its place.

Full Outer Join

 

 When two tables are joined using a Full Outer Join or simply Full Join all the rows from the left and right tables are displayed. However, the full outer join returns larger result sets.

 

The FULL OUTER JOIN returns all records when there is a match in either left (table1) or right (table2) table records.

Considering the example of Customers(table1) and Orders(table2) table, the full join returns all the data from the Customers table and Orders table apart from from returning the matching recoreds from both the tables. In case, there is no matching data in either of the table,full join returns a NULL in its place.
