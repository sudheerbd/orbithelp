Creating Active Directory Authority

 

To Create Active Directory Authority

To create a new authority

1.      On the User Management menu, click Authorities.

              

                       

2.      Authorities page appears with a list of authorities.   

      

3.     To create a new authority, click New (highlighted as "1") on the top right-hand side of the Authorities page.  

   

              

4.      New Authority dialog box appears.

  

   

 

5. In the New Authority dialog, enter the following to create a new authority

  • In the Name box, type the name of the authority.              

    •   Example: Name: AD Authority

  • In the Authority Key box, type the authority key.

    •  Example: Authority Key:AUTH_KEY

  • To sync the users/settings on login, select Sync on Login checkbox.

  • In the Description box, type the description for the authority.

  • Under the Authentication section, in the Authority Type box, type/select the required authority type from the following:

    • Oracle ERP Security

    • LDAP

    • Active Directory

    • People Soft

      • Example: Required Authority Type: Active Directory

    •  Based on the selected authority types, the fields in the Authentication section change.

  • In the Server URL box, type the URL of the Server.

    • Example : Server URL: ldap://192.168.2.126:389

      • Here, ldap is the protocol, "192.168.2.126" is the ip, and 389 is the default port number.

    • In the User DN box, type the Username.

    • Example: User DN: ORBIT8\Administrator

                In the Password box, type the password.

                 Example: Password: Welcome@123

          In the Base DN box, type the Base DN.  

Base DN: This is a point from where the server starts its search.

 Syntax: dc=orbit8,dc=com

  • When the user enters only Base DN, the server starts its search from the root, and retrieves all the users. The starting point (Base DN) is displayed here in the figure below (highlighted as "2"):
