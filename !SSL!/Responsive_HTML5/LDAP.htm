LDAP

Light-weight Directory Access Protocol (LDAP) is an open application protocol for accessing and maintaining distributed directory information services over an Internet Protocol (IP) network.

The figure below illustrates the structure of how the users and groups must be created in LDAP.

The initial search starts from the Base DN. The syntax of the Base DN is as follows:

?      Syntax: dc=orbit, dc=com

When the user enters only Base DN, the server starts its search from the root, and

retrieves all the users. Under the Base DN, Organizational Units(OU) are created in LDAP.

Here, the following two organizational units are created in LDAP:

·         ou=People (where ou is the attribute and People is its value)

·         ou=Roles (where ou is the attribute and Roles is its value)

 

Under the organizational units, users and roles are created. Here, under the ou – “People”, multiple users are created. And, under the ou- “Roles”, multiple roles such as Testing Role, Development Role, and so on are created. Once the structure is organized, the users/roles can be imported into Orbit.

In Orbit application, creating an LDAP Authority in the Authorities section under User Management, enables authorization and authentication to import users and roles into Orbit from LDAP.

Orbit enables authentication to import users from LDAP and authorization to import roles from authorities including Oracle EBS, PeopleSoft, Active Directory, and LDAP.
