Creating a PeopleSoft Authority

While creating PeopleSoft Authority, authorization and authentication enables to import users and roles into Orbit.

To Create a New PeopleSoft Authority

 

1.      On the User Management menu, click Authorities.

        

  2.      Authorities page appears with a list of authorities.   

 3.     To create a new authority, click New (highlighted as "1") on the top right-hand side of the Authorities page.  

4. New Authority dialog appears.

5. In the New Authority dialog, in the Name box, type the new authority name.

?   Example: Name: PS Form authentication

6. In the Authority Key box, type the required authority key.

?     Example: Authority Key: PSFORM

7. In the Description, enter the required description.

?   Example: Description: Test Description

 

Webservice-based Authentication

 

Webservice-based Authentication is the default way of Authentication

To perform webservice-based authentication:  

1. Under the Authentication section, in the Authority Type box, select the required option from the following:

·         Oracle ERP Security

·         Active Directory

·         LDAP

·         PeopleSoft

?  Example: Authority Type: PeopleSoft

2. In the URL, enter the webservice end connector URL.

?  URL: http://pshcm.orbit8.com:8000/PSIGW/PeopleSoftServiceListeningConnector/PSFT_HR

¥   For webservice-based authentication, providing data source is optional.

3. In the Data Source box, type/select the required data source.                                 

?  Example: Data Source: Peoplesoft

 ¥  Alternatively, create a new data source by clicking Create new datasource  

4. In the User Name box, type the required username.

?  Example: User Name: ORBIT_AUTHENTICATION

5. In the Password box, type the required password.

?  Example: Password: Laser2jet

For webservice-based authentication,  attributes are not required.

  • The username and password should remain same.

Form-based Authentication

 

To perform the form-based authentication

1. Under the Authentication section, in the Authority Type box, select the required option from the        

following:

·         Oracle ERP Security

·         Active Directory

·         LDAP

·         PeopleSoft

?   Example: Authority Type: PeopleSoft

 

2. In the URL, enter the webservice end connector URL.

?  URL: http://pshcm.orbit8.com:8000/psp/ps/?cmd=login

 

3. In the Data Source box, select/type the required data source.

?   Example: Data Source: Peoplesoft

 

4. In the User Name box, type the required username.

? Example: User Name: ORBIT_AUTHENTICATION

5. In the Password box, type the required password.

? Example: Password: Laser2jet

  • The username and password should remain same.

  • For form-based authentication, it is essential to mention the attributes.

To mention the attributes

1. On the Authentication section, click Attributes (highlighted as "1").

         

2. Attributes dialog appears.    

3. In the Attributes dialog, perform the following:  

a. To add a new attribute, Add Attribute.

b. Under Name, enter the required attribute.                       

c. Under Value, enter the required attribute.

 Required Attributes and their values

Attributes

 Values

formValidation

 true

validateFromTable

 true

 

   d. To save the changes, click Ok.

  •   The attribute names mentioned above should remain same.

Authorization

¥   Authorization same as Authentication checkbox is selected by default.  In case, the Authorization is not same as the Authentication, then remove the checkmark from the Authorization same as Authentication checkbox by selecting it.  This enables the Authorization area with two sections— Authority Type and Data Source. 

Adding a New Authority under Authorization

1. To add a new authority under authorization, click Add Authority.

2. Authorization dialog appears.

       3. In the Authority Type, select the required authority type.

¥ Here, the authority type does not need to be PeopleSoft essentially. The authority type in the Authorization section can vary from the authority type in the Authentication.

¥     Based on the selected Authority Type, the fields under it change.

4. To add a new attribute,  Attributes.

5. To save the changes in the Authorization section, click Ok.

 6. To save the changes, click Save.
