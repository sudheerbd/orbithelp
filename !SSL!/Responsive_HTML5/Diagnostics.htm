Diagnostics

Diagnostics section enables the user to view the logging levels of the following:

 

  • Web Application

  • Metadata Queries

  • ORBIT

 

Web Application : For web application server( for ex. Apache Tomcat), the  logging levels are available in log4j.properties file  

 

Metadata Queries:  

Turning on logging in Metadata Queries enables to :

  • monitor configuration details
  • facilitate debugging
  • view the SQL that is being sent to the database

 

ORBIT : In the ORBIT Analytics application, the log file with the log levels are defined under the Scheduled Processes in the Monitoring section.

 

Logging Levels

Web Application, Metadata Queries and ORBIT Analytics application provide log with the following logging levels.

 

Log Level Description
Off The OFF has the highest possible rank and is intended to turn off logging. This setting disables the generation of the log output. You may want to set logging to OFF during production to avoid the overhead of logging.
Error The ERROR level designates error events that might still allow the application to continue running.
Warning This level designates potentially harmful situations. This level enables logging of issues that have a potential to cause problems. For example, a setting that is picked by the application and not by the user.
Debug The DEBUG Level designates fine-grained informational events that are most useful to debug an application.  
Info The INFO level designates informational messages that highlight the progress of the application at coarse-grained level.  This level enables the standard output. The contents of this output is very limited. It is the default logging level if a logging level is not set.
Fine This level enables logging of the first level of the debugging information and SQL. You may want to use this log level during debugging and testing, but not at production.
All The ALL has the lowest possible rank and is intended to turn on all logging.  

 

 

## The logging information accumulates at every sequential log level: each log output includes the information from the output of the previous log level. For example, if you set the log level to INFO, your output will include log messages from INFO, DEBUG, WARNING and ERROR levels.
