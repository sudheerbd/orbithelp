 Setting Data Security

 

To set the Data Security in the Logical Model

 

1. Under Domain Hierarchy, right-click the required logical model, and then click Data Security.

 

2. Data Security- <Logical Model Name> dialog box appears.

 

    

 

                                            

3. Data Security has the following three options:

 

  • No Security

  • Global

  • Role-based

 

  •  To enable security, select Global or Role-based security and configure filters.

 

When No security is applied the complete data in the reports is visible to every user/role.

                                                                                                                                                                                               Go to Top

 

Setting Global Security

To set the Global Security

 

1. To set the global security, select the Global radio button.  

 

 

   

 

                                                   

2. The formula box (highlighted as "1") appears.

3.  To create a formula, click (highlighted as "2").

 

4. Expression Builder dialog box appears.

 

5. Create an expression with the following:

 

  • Category

  • Sub-Category

 

6. Category consists of the following:

 

  • Available Objects

  • Functions   

 

 

  • Sub-Category changes based on the selected Category

 

 

        

                                                                

 

 

7. To save the created formula, click Ok.

 

       

                                                                                                

 

  • When Global option is selected, the formula created and displayed here (highlighted as "3") must be added to the WHERE clause.

 

   

      Go to Top

 

 

Setting Role-based Security

To set the Role-based Security

 

 

1.  Under Domain Hierarchy, right-click the required logical model, and then click Data Security.

 

2. Data Security dialog box appears.

    

3. To set the role-based security, select the Role based radio button.

 

 

4. Role based filters area appears in Data Security dialog box.

 

5. To add a new role-based filter, click  Search for items to add (highlighted as "1").

 

     

                                        

 

4. A list of values appears on the right hand side as shown below:

 

     

                                                                                              

5. Select the required role from the available list.

 

6. The role appears under Role in Role based filters area.

                                                                                                      

7. To set the formula for the selected role, click in the Formula box.

 

 

8. Expression Builder dialog box appears.

 

 

9. Create an expression with the following:

 

  • Category

  • Sub-Category

 

10. Category consists of the following:

 

  • Available Objects

  • Functions   

 

 

  • Sub-Category changes based on the selected Category

 

 

  •  Example: Required Category : Available objects

                                                               

 

7. Build the required formula as shown above, and click Ok.

 

8. The formula appears as shown below:

 

      

                                                                                 

 

The formula is now assigned to the role "Analyst".  The value of grade id equal to 184 is visible to analyst role, provided the category  under required  business object has the "Analyst" Role assigned to it.

 

9. To save the changes, click Ok.

 

10. A message " Data security saved successfully" appears.

 

Assigning Analyst Role to the required BO

 

1. Open the required Business Object.

 

  • Example : Required BO: Role base

 

 

2. Under the Business Objects section, select the required business object (highlighted as "1") and categories (highlighted as "2"), and then assign the required role.

 

     Example: Required BO: Role base

                    Required Categories: PER Assignments

                                                    Per grades

                    Required Role: Analyst

3. To save the changes, click Save.

 

Creating a Report base on the Role-based Security

 

Open a user account which is assigned the "Analyst" role.

 

1. On Reports, click New Report, then click Data Report, and then click Ok.

 

 

 

2. Under Data Explorer, from the Search Business Object area , select the required business object.

 

  •    Example: Required Role:Role base   

 

     

 

3. From the selected business object, drag and drop the required columns, and then click Apply.
