Data Model

The figure below explains the data flow in the ORBIT application.

 

 

 The data is taken from the following data sources:

 

  • Database

  • Files

  • Web Services

 

The data from the data sources is pulled into the Physical Model. Physical Models and Logical Models are parts of a Domain. In the logical model, the relation between the data is established. This is where the Joins are created between the tables, views, or synonyms. Once a logical model is created, business object is automatically created.

 Using the business object, the following reports are created:

  • Data Reports

  • Analysis

  • Dashboards

  • Publisher Reports

  • Script Reports
