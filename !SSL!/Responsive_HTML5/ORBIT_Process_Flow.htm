ORBIT Process Flow

ORBIT application’s process flow is explained below.  

 

     

                          

                                                                                           Figure: ORBIT Process Flow

 

The figure shows a conceptual diagram of ORBIT process flow. Initially, you can create a business unit.  

 

The business unit in turn creates new users. In the Security section, users are created; they are grouped and assigned roles. Using Authorities feature, you can import users from multiple databases, including Oracle JDBC, SAP MaxDB, SyBase, Firebird, and other databases. In the Authorities section, you can sync users and roles. In the Settings section, you can configure Email settings. Permissions are set on roles. Using Permissions, you can customize the available features based on the roles.  Once the Security is handled, you can create a new data source. For the selected data source, you can create a Domain. In the Domain section, you can create Physical Model, Logical Model and Business Object.  You can assign the roles and users for a specific business object in BO security section.

 

You can create a report based on the available business object. Currently, ORBIT application provides three types of reports— Operational Reports, Analysis, and Dashboards. Using Meta Objects, you can apply conditional formatting to the available reports.
