Introduction

 

Pre-requisites

 

The minimum pre-requisites for Discoverer migration into ORBIT Analytics application are as follows:

 

·    Configure the required data sources

· In the <administration.properties> file, set the database details of EUL.

 

User must be aware of EEX file export procedure in Discoverer Administrator. To import the EEX file into the ORBIT application, the user must have the exported EEX File.

 

   Using ORBIT Discoverer Migration, discoverer reports are imported into ORBIT Analytics application.

 

          

 

 

The figure above depicts the conceptual diagram of the migration from ORACLE Discover to ORBIT Analytics application.

 

Initially, business areas are migrated, where in the required business areas are selected. The metadata that defines one or more business areas is downloaded as an XML file. This xml file is imported into the ORBIT application using the Monitoring section. Once the xml file is uploaded, a domain is created for the selected business areas under Domains in ORBIT application.

In the ORBIT application, assign the BO security to the migrated business object. Once the BO security is applied, create reports and view reports under Discoverer Reports Folder.

Alternatively, reports can also be migrated into ORBIT application by migrating the worksheets under a specific business area.  In the worksheet migration process, export the report metadata using Discoverer Desktop administrator. When the report metadata is exported, it creates an EEX file.  This EEX file is imported into ORBIT application’s Monitoring section. The migrated reports appear under Discoverer Reports Folder in the Reports section.
