Custom job API

 

The OrbitJobContext is the primary job context which provides many details related Orbit Jobs.

 

  1. getLogger()

 

 It is ORBIT logger, which is used to log debug/error messages.

 

Implementation Code :

 

 

package com.orbit.reporting.custom.job;

import java.io.File;

import org.quartz.JobExecutionContext;

import org.quartz.JobExecutionException;

import com.orbit.reporting.job.OrbitJobContext;

import com.orbit.reporting.job.OrbitQuartzJob;

public class MyCustomJob extends OrbitQuartzJob {

      @Override

public void execute(JobExecutionContext context) throws JobExecutionException

     {

         try{

OrbitJobContext ctx = getOrbitJobContext(context, this);

         ctx.getLogger().debug("Entered into MyCustomJob...");

         File sorceFilePath = new 

File(ctx.getFilePath("murty.txt"));

         ctx.generateOutputFile(sorceFilePath);     

ctx.getLogger().debug("execution is completed.....");

         }

   catch(Exception ex){

                 ctx.getLogger().error("error : "+ex.toString());

}

      }                                

}

 

  
           Similarly, implementation code for other below-mentioned methods can also be written.

 

  1. generateOutputFile()

 

It is used to generate output file based on output type selected from ORBIT User Interface.

 

  1. getFilePath()

 

It is used to get ORBIT configured output directory where output files are generated.

 

  1. getJobParams()

 

When you register a custom job, you can send couple of parameters from ORBIT UI as per custom job. You can retrieve those parameters using this method.

 

  1. getParentProcessId()

 

Every ORBIT job has a unique process ID. This method is used to get the process ID.

 

  1. getOutputType()

 

This method is used to get the output type configured from ORBIT User Interface.

 

  1. getOutputFile()

 

This method is used to get the output file, if generated.

 

  1. getOutputStream()

 

It is used to push any output content into this output stream. It is required to close this stream when a job is completed. It is part of the job implementation.

 

  1. getJobId()

 

This is to get the custom job id

 

  1. getJobConfig() 

 

This method is to get the value entered in the job config parameter in the UI.  

 

  1. getParameterType(parameter)

 

This method is to get the parameter type defined in the UI either STRING,DATE, or INTEGER.
