Setting Logical Model Properties

To set the properties for Logical Model

1.      Under the Domain Hierarchy, Logical Models folder appears. 

2.      In the Logical Models folder, right-click the required logical model, and then click Properties.

 

        

                         

3.      Properties- <Logical Model Name> dialog box appears.

 

       

 

  1. The selected logical model appears under Search box on the left hand side (highlighted as "1")  with its associate table/view/synonym under it (highlighted as "2").

  2. The properties of the logical model, table/view/synonym and business objects appear on the right-hand side.

 

Properties for Logical Model

 

1.      Under Logical Model, perform the following:

a.       Under the Name field, type the name of the logical model.

b.      Under the Description field, type the description for the logical model.

c.    Under the Query tab, perform the following:

        i. In the Delay Outer Join box, type/select the required option from the following:

        • true

        • false

      ii.  In the Query Executor box, type/select the required option from the following:

 

a. Default - option across any database connection / query to be built on Orbit. Internal algorithm differentiates different databases and builds appropriate SQL.

b. ORACLE ERP - this option is used specifically for the discoverer conversion domains in conjunction to the EBS security, where the reports are shared to specific responsibilities. For non-EBS based discoverer reports, query executor changes appropriately. Reports built on domains with this executor have a default parameter of "Responsibility" as mandatory picker.

c. ORBIT Content - this option used for most of the EBS content built in Orbit that utilizes the Oracle EBS user, responsibility, application, and security group from EBS to initialize apps and there after run the report. This options also incorporate cases for GL related accounts, sets of books etc. Reports built on domains with this executor have a default parameter of "Responsibility" as mandatory picker.

d. Default (with prompt init) - this option takes in to account the user input values as report parameters (some cases from discoverer) [with option of No Condition] that act as feeders to the database view or physical model or logical model as column, condition, calculation, or expression.              

e. SQL Inline Param                                              

For different datasource like Athena, select physical model miscellaneous as SQL Inline Param.  

d.        Under the Miscellaneous tab, perform the following :

          i. In the GL Accounts LOV box, type/select the required option from the following:

        •   Optional

        •   Required

  Properties for Table/Synonym/view

1.      For the required table/view/column, perform the following:

a.   Under the Name field, type the name of the table/view/synonym.

b.   Under the Description field, type the description for the table/view/synonym.

c. Under the Table tab, perform the following:

 i.  In the Table Type box, type/select the required option.

 ii.  Target Schema is automatically picked up from the data source. 

iii.    In the Target Table box, type the target table or select the target table textbox, and then click .

iv.      In the Target Table Type box, type/select the required option.

d. Under the Miscellaneous section, perform the following:

  i.  In the Relative Size box, enter/edit the required relative size.

Relative size can be assigned at the Physical Model level, logical model level and in the logical tables at Business object level. Refer to Setting Relative Size.

e.  Under the Refresh Options section, perform the following:

      1. In the Refresh File Pattern box, select the required option.

        • Recently Uploaded

        • Sort By File Name

      1. In the Refresh Type box, select the required option.

        • Full Load

        • Incremental

2. To save the changes, click OK.

3.  To close the dialog box, click Cancel.

 

Properties for Business Object

1.      For the required business object, perform the following:

a. Under the Name field, type the name of the business object

b. Under the Description field, type the description for the business object.

2.      To save the changes, click OK.

3.      To close the dialog box, click Cancel.
