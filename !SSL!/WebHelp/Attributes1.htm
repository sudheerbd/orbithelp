Attributes

All the Active Directory (AD) objects have attributes that take unique or multiple values, these values describe the object characteristics. For example, a user object in Active Directory will have attributes such as his first name, last name, email etc.

The name of an attribute is similar to the name of a field in a database. The actual value assigned to the attribute is stored in AD.

Orbit supports the attributes with the following values:

  • userNameAttributeValue = “sAMAccountName”

  • groupNameAttributeValue = “cn”

  • groupRoleAttributeValue = "memberOf”

  • userObjectClassValue = “user”

However, the available attributes can be overwritten using the required attribute value. 

To Change the Attributes

  1. In the Edit Authority dialog, on the Authentication section, click  Attributes.

  2. The Attributes dialog appears.

  1. Using the Attributes section, add the new attribute or delete the existing attribute.

  2. To add a new attribute, click  Add Attribute.

   

  1. By default, the “userNameAttributeValue” has a value “sAMAccountName”. However, if there is a need to overwrite the userNameAttributeValue, then this can be done using the attributes.
