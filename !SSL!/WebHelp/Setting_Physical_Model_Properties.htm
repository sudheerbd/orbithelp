Setting Physical Model Properties

To set the physical model properties

 

  1. Under Domain Hierarchy, Physical Models folder appears.  

  2. Under the folder, right-click the required folder, and then click Properties.

     

        

  1. Properties-<Physical Model Name> dialog box appears.

 

       

  1. The selected physical model appears under Search on the left hand side (highlighted as “1”) with its associate table/view/synonym under it.

  2. The properties of the physical model and table/view/synonym appear on the right-hand side (highlighted as “2”).

 

Properties for Physical Model

 

  1. Under Physical Model, perform the following:

    1. Under the Name field, type the name of the physical model.

    2. Under the Description field, type the description for the physical model.

    3. Under the MetaObject tab, perform the following:

      1. In the Base MetaObject box, type the name of the base meta-object or select the textbox and then click.

Select MetaData Object dialog box appears.

  1. In the Boolean MetaObject box, type the name of the boolean meta-object or select the text box and then click.

Select MetaData Object dialog box appears.

  1. In the Currency MetaObject box, type the name of the currency metaobject or select the text box, and then click.

Select MetaData Object dialog box appears.

  1. In the Date MetaObject box, type the name of the date meta object or select the text box, and then click

Select MetaData Object dialog box appears.

  1. In the Id MetaObject box, type the name of the Id meta-object or select the text box, and then click.

Select MetaData Object dialog box appears.

  1. In the String MetaObject box, type the name of the String meta-object or select the text box, and then click.

Select MetaData Object dialog box appears.

  1. In the Time MetaObject box, type the name of the Time meta-object or select the text box, and then click .

Select MetaData Object dialog box appears.

    1. Under the Miscellaneous tab, in the Query Executor box, type/select the required option from the following:

      1. Default - option across any database connection / query to be built on Orbit. Internal algorithm differentiates different databases and builds appropriate SQL.

      2. ORACLE ERP - this option is used specifically for the discoverer conversion domains in conjunction to the EBS security, where the reports are shared to specific responsibilities. For non-EBS based discoverer reports, query executor changes appropriately. Reports built on domains with this executor have a default parameter of "Responsibility" as mandatory picker.

      3. ORBIT Content - this option used for most of the EBS content built in Orbit that utilizes the Oracle EBS user, responsibility, application, and security group from EBS to initialize apps and there after run the report. This options also incorporate cases for GL related accounts, sets of books etc. Reports built on domains with this executor have a default parameter of "Responsibility" as mandatory picker.

      4. Default (with prompt init) - this option takes in to account the user input values as report parameters (some cases from discoverer) [with option of No Condition] that act as feeders to the database view or physical model or logical model as column, condition, calculation, or expression.  

      5. SQL Inline Param. 

For different datasource like Athena, select physical model miscellaneous as SQL Inline Param.  

    • The Option in the Query Executor can be applied both in the Physical Model properties and Logical Model Properties. If applied at Physical Model, the option holds good at its corresponding Logical model and business objects too.

    • If the option is applied at Logical Model, then the option is applied only to its business objects.

 

Properties for Table/View/Synonym

To view table/view/synonym

 

  1. For the required table/view/column, perform the following:

 

 

    1. Under the Name field, type the name of the table/view/synonym.

    2. Under the Description field, type the description for the table/view/synonym.

    3. Under the Table tab, perform the following:

      1. In the Table Type box, type/select the required option from the following:

        • Dimension

        • Fact

        • Other

      2. Target Schema is automatically picked up from the data source. 

      3. In the Target Table box, type the target table or select the target table textbox, and then click.

      4. In the Target Table Type box, type/select the required option from the following:

        • Table

        • View

        • Synonym

        • Inline_SQL

    1. Under the Miscellaneous section, perform the following:

      1. In the Relative Size box, enter/edit the required relative size.

Relative size can be assigned at the Physical Model level, logical model level and in the logical tables at Business object level. Refer to Setting Relative Size.

    1. Under the Refresh Options section, perform the following:

      1. In the Refresh File Pattern box, select the required option.

        • Recently Uploaded

        • Sort By File Name

      1. In the Refresh Type box, select the required option.

        • Full Load

        • Incremental

  1. To save the changes, click OK.

A success message appears as "Properties saved successfully".

  1. To close the dialog box, click Cancel.
