Creating a SAML Authority

To Create a SAML authority

 

  1. On the left-hand side menu, point to  User Management, and then click  Authorities.

  2. Authorities section appears with a list of available authorities.

  3. On the Authorities section, click  (highlighted as “1”).

 

  1. New Authority dialog box appears.

  2. In the New Authority dialog box, perform the following:

a.  In the Name box, type the name of the authority.

Authority Key: SAMLSAMP

b. In the Authority Key box, type the authority key.

Authority Key: SAMLDS

c. In Description box, type the description.

Description: This is sample description

d. Under the Authentication section, perform the following:

      1. From the Authority Type list, select the required authority.

Required authority: SAML

      1. To Import Idp metadata, user have two options:

  • URL

  • File   

      1. To Import Idp Metadata using url, select URL radio button.

      2. In the URL, type the required IDP metadata URL.

IDP Url: https://dev-174968.oktapreview.com/app/exkh1nabygp3NGFmC0h7/sso/saml/metadata

This URL is same as the URL of the Identity Provider metadata in IDP site.

        1. To Import Idp Metadata using file, select File radio button.

        2. In the File, click Upload next to the text field.

        3. Select the required idp metadata file, from the local machine .

The file is an xml, provided by IdP like IDCS, ADFS etc.

        1. Once the file data uploaded, IdP Entity Id will appear

  1. To save the changes, click Save.

Once the authority is created, it appears under Authorities section (highlighted as “2”).

 

    

 

  1. Now logout and login again using the following URL:

URL: https://orbdev01.orbit8.com/reporting/

Here, https, orbit tenant have been used since the SAML authority is now configured in ORBIT application.
