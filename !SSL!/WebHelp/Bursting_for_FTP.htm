Bursting for FTP

To perform bursting for FTP

  1. In the Delivery Plans dialog box, perform the following:

    1. In the Plan Name box, type the required delivery plan name.

      Plan Name : Bursting

    2. In the Description box, type the required description.

      Description is an optional field.

    3. In the Delivery Type box, select the required delivery type from the following:

      • EMAIL

      • FTP

      • FILE

    4. To maintain the history of the delivery plan, select the Maintain History checkbox.

    5. In the Split/Delivery By box, drag the required field from the business object section.

      Since, bursting is done region-wise, drag Region from the business object section.

      Split/Delivery By: Region

    6. In the Report Mapping Field list, select the required field that needs to be mapped to the field with which the bursting is done.

      Since, bursting is done region-wise, and “Sample-Superstore”.
      “Region” appears in the Split/Delivery By box, the Report Mapping Fields must be mapped appropriately.
      Split/Delivery by and Report Mapping Fields are mandatory fields.

      Report Mapping Field: Region

  2. Under FTP tab, perform the following:

    1. In the Select Server box, drag and drop the required field from the business object or select the already available FTP server from the list.

      Here, the existing FTP details have been selected.

      Select Server: Test Bursting

    2. In the Server Name box, drag and drop the required field from the business object or type the required server name by clicking .

    3. In the User Name box, drag and drop the required field from the business object or type the required user name by clicking .

    4. In the Password box, drag and drop the required field from the business object or type the required password by clicking .

    5. In the Port box, drag and drop the required field from the business object or type the required port by clicking .

    6. In the Remote Directory box, drag and drop the required field from the business object or type the required remote directory by clicking .

    7. In the Remote File Name box, drag and drop the required field from the business object or type the required remote file name by clicking .

      Server Name, User Name, Password, Port, Remote Directory, and Remote File Name depend on the desired business environment.

    8. From the Secured list, select the required option from the following:

      • false

      • true

    9. From the Zip list, select the required option from the following:

      • false

      • true

    10. In the Password box, drag and drop the required field from the business object or type the required password by clicking .


      Password protection is applicable only for PDF, Excel and Word.
      Password is case sensitive.

  4. To cancel any updates, click Cancel.
