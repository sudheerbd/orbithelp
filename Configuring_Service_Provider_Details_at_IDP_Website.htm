Configuring Service Provider Details at IDP Website

To configure the service

 

  1. Navigate to Identity Provider website.

  

   

  1. In the Identity Provider website, perform the following:

    • In the Username box, enter the required user name.

    • In the Password box, enter the required password.

These username and password will be created by the IDP admin

  1. Upon login, User Home page appears.

  2. On the right-hand side, click Admin.

 

    

  1. Dashboard page appears.

  2. In Dashboard page, on the top right-hand side of the page, click Developer Console list.  

    

 

  1. From the list, select Classic UI.

 

  1. In the Classic UI view, in the Dashboard page, under Shortcuts, click Add Applications.

 

  1. In Add Applications page, on the left-hand side, click Apps you created.

  

 

  1. On the right-hand side, for the selected application, click Edit.  

To create a new application, click Create New App.

  

 

  1. For the selected application, edit the following:

    • Under Edit SAML Integration section, in the General Settings, enter the required fields and then click Next.

   

    • Under Edit SAML Integration section, on the Configure SAML tab, in the SAML Settings, in the General section, perform the following:

      1. In the Single sign on URL box, enter the required SSO URL.

https://orbit.localhost:8443/reporting/saml/SSO

After application URL, add saml/SSO for single sign on URL.

      1. In the Audience URI (SP Entity ID) box, enter the metadata URL 

https://orbit.localhost:8443/reporting/saml/metadata

      1. In the Default RelayState box, enter the relay state.

OKTA_RELAY_STATE

      1. In the Name ID format list, select the required option from the following:

        • Unspecified

        • EmailAddress

        • X509SubjectName

        • Persistent

        • Transient

      1. In the Application username list, select the required option from the following:

        • Okta username

        • Okta username Prefix

        • Email

        • Email prefix

        • Custom

        • None

Required Application username: Okta username

 

 

  1. To view the advanced settings, click Show Advanced Settings.

  2. Under the advanced settings, perform the following:

  3. In the Response list, select the required response from the following options:

    • Signed

    • Unsigned

Response: Signed

  1. In the Assertion Signature list, select the required signature list from the following options:

    • Signed

    • Unsigned

  1. In the Signature Algorithm list, select the required algorithm from the following options:

    • RSA-SHA256

    • RSA-SHA1

  1. In the Digest Algorithm list, select the required algorithm from the following options:

    • SHA256

    • SHA1

  1. In the Assertion Encryption list, select the required encryption type from the following options:

    • Encrypted

    • Unencrypted

  1. To logout from the application after single sign-on, enable Single Logout URL by selecting Allow application to initiate Single Logout checkbox.

  2. In the Single Logout URL box, enter the required application Logout URL. 

Logout URL: https://orbit.localhost:8443/reporting/saml/logout

The application URL should be followed by “/saml/logout”

  1. In the SP Issuer box, enter the issuer for the service provider.

SP Issuer: https://orbit.localhost:8443/reporting/saml/metadata

  1. To enter the Signature certificate, click Browse.

  2. The path of the signature certificate appears in the textbox before Browse.

  3. To upload the certificate, click Upload.

Prior to uploading the certificate, you must have a valid SSL certificate.

  1. In the Authentication context class list, select the required option from the list.

Authentication context class: PasswordProtectedTransport

  1. In the Honor Force Authentication list, select the required option from the list.

 HFA: Yes

  1. In the SAML Issuer ID box, enter the SAML IDP issuer ID. 

 http://www.okta.com/${org.externalKey}

 

Configuring Attributes

To configure attributes

 

  1. In the ATTRIBUTE STATEMENTS(OPTIONAL) section, enter the required attributes.

Here, Firstname, Lastname, Email, Login are the required attributes as recommended by ORBIT to create a New App.  

  1. In the GROUP ATTRIBUTE STATEMENTS(OPTIONAL) section, enter the required attribute.

 

  1. To navigate to the next section, click Next.

  2. This navigates to Feedback tab.

  3. In the Feedback section, provide the required feedback, and then click Finish.

 

  1. On the Sign On tab, under Settings section, click Identity Provider Metadata.

  

  1. An xml file appears.

  2. In the xml file, select the IDP metadata URL.

  1. This IDP metadata URL must be used in the Authorities section in ORBIT application.
